Yonder Design System

A unified visual language for the Yonder social media intelligence platform. Built for speed, clarity, and consistency across every screen analysts interact with daily.

1

Principles

🎯
Signal over noise
Every component prioritises actionable intelligence. Visual hierarchy always points analysts to what needs attention first.
⚡
Fast & fluid
Transitions at 180ms. Hover feedback on every interactive element. The interface responds before the analyst has time to wonder.
🧩
Composable
Tokens, not one-offs. Every color, shadow, and radius is a CSS custom property so the whole system can be re-themed in minutes.
2

Token Architecture

All values live in CSS custom properties on :root. Components reference tokens — never raw values — so a single change propagates everywhere.

/* Brand */
--p: #7c3aed;  --pd: #6d28d9;  --pl: #ede9fe;
/* Semantic — Impact */
--red: #ef4444;  --amber: #d97706;  --blue: #2563eb;
/* Surfaces */
--bg: radial-gradient(ellipse at 20% 20%, #fce7f3 0%, #cdb6ec 100%);
--card-shadow: 0 1px 3px rgba(80,0,120,.06), 0 4px 16px rgba(80,0,120,.07);
--card-r: 12px;  --f: 'Heebo', sans-serif;  --e: .18s ease;

Impact Color Usage

The red/amber/blue impact system must be used consistently. An analyst who sees red anywhere in the UI should immediately associate it with High impact — never use red for decorative purposes.

High Impact Moderate Impact Low Impact
Do

Use red, amber, and blue exclusively for impact levels. Analysts rely on color memory to scan cards quickly.

Don't

Don't reuse impact colors for unrelated UI actions (like delete or save). Reserve them exclusively for the impact system.

2

Accessibility

WCAG AA — All text/background combinations must meet 4.5:1 contrast ratio. Primary purple (#7c3aed) on white achieves 6.2:1. White on impact red (#ef4444) achieves 4.6:1. Never render small text using --faction-c on white — it fails at 2.9:1 and must be used at 14px+ bold only.

Buttons

Buttons initiate actions. Every button variant communicates a distinct level of emphasis — from the gradient-filled primary for the most critical action to the ghost outline for secondary choices.

Overview
Guidelines
Specs
1

When to use buttons

Primary — one per screenUse for the single most important action: "Sign in", "Apply Filters", "Save Conversation". Its gradient fill commands immediate attention.
Outline — secondary actionsPaired alongside a primary button for cancel, reset, or go-back actions. The purple border keeps it branded without competing for focus.
Tonal — low-emphasis actionsPale purple background for actions that need to be visible but shouldn't compete with primary CTAs — e.g., "+ Add Narrative" inside a curate drawer.
Destructive — irreversible actionsRed tint for actions like "Delete Faction" or "Clear All Alerts". Always pair with a confirmation dialog before executing.

1

Emphasis hierarchy

Screens should have at most one primary button. Supporting actions use outline or tonal variants. Stacking two primary buttons removes the hierarchy that makes them effective.

Do

Pair one primary with one outline. The hierarchy is clear — apply is the main action.

Don't

Don't use two primary buttons side by side — analysts can't tell which action is more important.

Do

Don't

Don't shrink a default button with inline styles — use .btn-sm instead for consistent sizing.

Tabs

Tabs organise narrative detail content into four analytical views. They appear as a pill group in a gray tray, with the active tab lifted to a white surface to signal selection.

Overview
Guidelines
Specs
1

Narrative detail tabs

The four tabs in narrative detail each answer a distinct analyst question. This order is fixed — analysts develop muscle memory for which tab answers which question.

Who is Involved?
Shows a stacked area chart of faction engagement over time. Identifies which groups are amplifying the narrative and at what relative proportion.
Should I care?
Displays an impact spectrum bar and per-faction inauthenticity scores. Helps the analyst quickly decide whether a narrative warrants escalation.
Why is happening?
Line chart of narrative volume over time with annotated inflection points. Surfaced AI analysis explains probable trigger events.
What can I do?
Recommended response actions with priority indicators — monitor, respond, escalate. Links to relevant team workflows.
1

Tab ordering

Fixed order is required. The four narrative tabs must always appear in the sequence: Who → Care → Why → What. This mirrors the analyst's cognitive workflow: identify actors → assess risk → understand cause → choose response. Reordering breaks the mental model analysts have built.
2

First tab always active

Do

Always open the narrative detail view with the first tab ("Who is Involved?") active.

Don't

Don't persist the last-active tab across navigation — always reset to "Who is Involved?" on entry.

Brand & Identity

The Yonder brand mark is an 8-pointed asterisk (×) — four lines intersecting at 45° intervals, drawn in purple. It never appears in raster form; always as inline SVG so it scales perfectly to any size.
